Desafíos adicionales

  1. ¿Por qué no deberíamos repetir el id en varios elementos?

    2. El atributo id en HTML debe ser único dentro de una página, ya que sirve para identificar un solo elemento del DOM de manera precisa. Cuando se repite el mismo id en varios elementos, se rompe este principio de unicidad, lo que puede causar problemas en el comportamiento del código, como:

    JavaScript:Si intentas seleccionar un elemento con document.getElementById("destacado"), solo devolverá el primer elemento con ese id, ignorando los demás, lo que puede generar errores de funcionalidad.

    CSS: Los selectores de id tienen una alta especificidad, y si se repite en varios elementos, es impredecible cómo los navegadores aplicarán los estilos, ya que no están diseñados para manejar esta situación correctamente.

  2. ¿Qué tiene más peso, el id o la clase en el mismo elemento?

    3. En CSS, los selectores tienen diferentes niveles de especificidad. El id tiene más peso que la clase. Esto significa que, si un elemento tiene tanto un id como una clase, los estilos definidos por el id se aplicarán con mayor prioridad sobre los estilos definidos por la clase. Por ejemplo, en el párrafo con id="especial" y class="contenido", aunque ambos selectores aplican estilos, los estilos del id tendrán mayor peso. Sin embargo, si defines estilos más específicos (como combinando el id y la clase: #especial.contenido), estos sobrescribirán ambos.

  3. Comportamiento en el navegador

    4. Clase contenido: Aplica color de texto y márgenes a los párrafos.

    Id especial: Cuando ambos (clase e id) están presentes, el id tiene más peso para estilos como el fondo y el borde.

    Esta es la razón por la que puedes sobrescribir el estilo del párrafo especial combinando tanto el id como la clase en tu CSS.
